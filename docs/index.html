Trentino Territory & Tourism Facilties

Dongili Lorenzo, Grotto Giulia, Sacchet Stefano

View Project

Purpose

This project discusses the development of a Knowledge Graph which aim is to provide an easy and convenient access about key tourism facilities in the region, such as hotels, restaurants, museums, natural parks, and other attractions. By integrating various knowledge and data resources, this KG will serve as a foundational tool for applications and services focused on delivering comprehensive and user-friendly information about the Trentino region. In a nutshell, the purpose of this project is described as:

"A service designed to inform users about various tourist facilities available in Trentino.”

ER Model

The gathered knowledge and data resources have been utilized to develop the Entity Relationships model. This diagram summarizes the key aspects to be addressed by the knowledge layer of the final Knowledge Graph and has guided the subsequent stages of development.

Data Resources

For hotel and ski slope data, resources from DatiOpen.it were utilized. Climbing route data was obtained from Kaggle , while information on hiking trails, rental shops, restaurants, cycle routes, and viewpoints was sourced from OpenStreetMap (OSM). The data underwent thorough cleaning and standardization through Python scripts, during which additional properties such as coordinates and elevations were incorporated. Finally, the processed data was consolidated into CSV files.

Knowledge Resources

Given the crucial role of schemas in knowledge and data integration, they were used as the foundational knowledge resources. To ensure reusability and provide a robust structure for our model, we gathered the data from OpenStreetMap (OSM) and adhered as closely as possible to its schemas. This approach was taken to enhance alignment with existing standards and improve future reuse. Specifically, the following references from OSM were incorporated:

  • Ski Rental
  • Cycle Routes

From schema.org the following references were incorporated:

  • Hotels

ETG

Check this

The generation of the Entity Type Graph (ETG) was achieved by combining the teleontology and intermediate ontology developed during the previous phases using the Protégé development tool. This process involved modifications such as reorganizing entity types, resulting in a newly formalized ETG schema. To finalize the ETG, entities were sanitized to ensure the use of unambiguous identifiers, enhancing the schema's future reusability. The KOS tool was employed for language alignment, assigning unique Global Identifiers (GIDs) to each entity type in accordance with the Universal Knowledge Core (UKC) standard.

Knowledge Graph

The final stage involves the creation of the EG from the ETG and the refined data provided by Karmalinker. The resulting Knowledge Graph consists of 6 entity types, 21 data properties, and 5 object properties: has_calendar, has_lines, has_schedule, has_stops, and has_trips. This final KG has been used to answer the Competency Queries through SPARQL queries.
